ガチャ カスタムアソート シミュレータ v2 解説

対象ファイル：gacha_sim_custom_upload_v2.html ｜ HTML / CSS / JavaScript のみで動作する単一ファイルアプリ

1. このアプリは何をするもの？

自分のPCにある画像（サムネイル）をアップロードして、ガチャガチャのアソート（中身の内訳）を再現し、 「実際に回したらどの順番で何が出るか」を疑似体験できるシミュレータです。

  • 景品の種類数（1〜24種）と、それぞれの初期数・排出済み数を自由に設定できる
  • 各タイプに自分の画像を割り当てられる（例：実際の商品のサムネイル写真）
  • ガチャボタンを押すたびに、残っている中からランダムに1個が出る（＝現実のガチャと同じ「引いたら減る」方式）
  • 残り数のバーグラフ・次に出る確率・引いた順の履歴がリアルタイムに更新される

v2 で追加された機能

機能内容
NEW設定の自動保存種類数・名前・初期数・排出済が入力のたびに自動保存され、次回開いたとき復元される
NEW画像の自動縮小アップロード画像を長辺360px（表紙は600px）に自動縮小してから保存。大きい写真でも容量エラーになりにくい

ポイント：完全ローカル動作。 サーバーへの送信は一切ありません。アップロードした画像・設定はブラウザ内の保存領域（localStorage）に記録されるだけで、 PCの外には出ません。ファイルをダブルクリックで開くだけで動きます。

2. 画面構成

画面は左右2パネル構成です。左で設定 → 右でシミュレートという流れになっています。 左右のパネルはそれぞれ独立してスクロールできるため、種類数が多くても 片側の操作でもう片側の表示位置がずれません。

◀ 左パネル：アソート設定 ・表紙画像のアップロード
・種類数の数値入力（1〜24種）
・タイプごとの行：
　画像アイコン／名前／初期数／排出済／残り
・合計表示（初期・排出済・残り）
・「シミュレート開始 ▶」ボタン
シミュレーション画面 ▶ ・結果表示キャンバス（280×280px）＋ガチャボタン
・「10連 ▶」「↩ 1回戻す」ボタン
・タイプ別の残り数バー＋確率%
・引いた履歴グリッド＋「CSV保存」
・全部出たら「【全商品終了】」バナー

左パネルの各項目

項目内容
表紙画像ガチャを引く前にキャンバスに表示される画像。商品パッケージの写真などを想定。未設定ならグレーの案内表示になる。
アソート種類数1〜24の数値を入力。入力した数だけタイプ行（A〜X）が表示され、非表示のタイプは計算にも抽選にも含まれない。範囲外の値は自動で1〜24に補正される。
画像アイコン（●）クリックするとファイル選択が開き、そのタイプのサムネイル画像を割り当てられる。割り当て済みはアイコンに画像が表示される。
名前タイプの表示名（最大12文字）。バーグラフのラベル・履歴のバッジ・CSVに反映される。
初期数そのタイプがアソートに入っている個数（0〜999）。
排出済すでに現実で出てしまった個数。「初期数 − 排出済 ＝ 残り」が抽選対象になる。途中からのシミュレートができるのはこの欄のおかげ。
残り自動計算（入力のたびに即時更新）。手入力はできない。

3. 抽選の仕組み（いちばん大事なところ）

このアプリの抽選は「確率○%で当たり」という確率方式ではなく、 現実のガチャマシンと同じ「箱の中から取り出す」方式（非復元抽選）です。

① プール生成
タイプごとに「残り個数」分のカプセルを配列に詰める
（例：A残り7個 → Aのカプセルを7個入れる）
▼
② シャッフル
フィッシャー–イェーツ法で配列を偏りなくかき混ぜる
▼
③ 抽選
ボタンを押すたび、配列からランダムな位置の1個を
splice で「取り除いて」結果にする
▼
④ 更新
残り数・確率%・バーグラフ・履歴に反映。
プールが空になったら「全商品終了」
取り除く方式なので：同じ個体は二度と出ない／残りが少ないタイプほど出にくくなる／ 最後まで引けば必ず設定した内訳どおりの結果になる——という現実のガチャの性質がそのまま再現されます。

10連ガチャの動き

  • 内部的には1回引く処理を最大10回繰り返しているだけ。特別な確率補正はない
  • 残りが10個未満のときは引けるだけ引く（例：残り4個なら4連で終了）
  • キャンバスには最後に出た1個を表示し、履歴には10個すべてが順番に追加される

「1回戻す」（アンドゥ）の動き

  • 直前に引いた1個をプールに戻し、残り数・カウント・履歴の最後のコマを巻き戻す
  • 売り切れ後でも戻せる（「全商品終了」バナーも消えてガチャ再開できる）
  • 連打すれば何回でも遡れる（履歴が空になるまで）。戻した後に引き直せば、当然結果は変わる
  • 次に出る確率は「残り÷全体」で計算しているので、戻した瞬間に確率表示も正しく戻る

次に出る確率の表示

各バーの右側の「残り 3/7｜15.0%」は、左が残り数／シミュレート開始時の数、右が次の1回でそのタイプが出る確率です。 確率は「そのタイプの残り数 ÷ 全タイプの残り合計」で毎回計算し直しています。

入力チェック（バリデーション）

  • 排出済みが初期数を超えているタイプがあると、エラー表示して開始しない
  • 全タイプの残りが0（引ける物がない）の場合もエラー表示して開始しない

4. 画像アップロードと自動縮小の仕様

アップロードの流れ

  1. アイコンをクリック → ファイル選択ダイアログで画像を選ぶ
  2. FileReader が画像をデータURL（Base64文字列）に変換する
  3. NEW長辺が規定サイズを超える画像は自動縮小（タイプ画像：360px、表紙：600px）。 PNGは透過を維持したままPNGで、それ以外はJPEG（品質85%）で再圧縮する
  4. 縮小後のデータを localStorage（ブラウザ内の保存領域）に書き込む
  5. アイコンのサムネイル表示を更新する

スマホで撮った数MBの写真でも保存時は数十KB程度になるため、 24種すべてに写真を割り当てても localStorage の容量（約5MB）に収まります。

画像の優先順位（3段フォールバック）

  1. アップロードしたカスタム画像（あれば最優先）
  2. なければ同フォルダの images/a.png 〜（デフォルト画像）
  3. それも読めなければタイプ色のカラーパネル＋英字（例：茶色地に大きく「A」）

つまり画像を1枚も用意しなくても、色分け表示で最後まで動作します。

5. データ保存の仕様（localStorage）

保存されるのは次の3種類。すべて閲覧者自身のブラウザの中だけに保存されます。

キー保存内容
gachaCustomImg_a 〜 gachaCustomImg_xタイプA〜Xに割り当てた画像（縮小済みデータURL）。旧版と共通キーなので旧版の画像も引き継がれる
gachaCoverImg表紙画像（縮小済みデータURL）
gachaAssortSettingsNEWアソート設定一式（種類数と、全タイプの名前・初期数・排出済）のJSON
  • 設定は入力のたびに自動保存。保存ボタンはない
  • ページを開くと loadSettings() が前回の設定を復元し、続けて画像も復元される
  • 引いた履歴（何回目に何が出たか）は保存されない。「シミュレート開始」のたびに白紙から始まる。記録を残したいときはCSV保存を使う
注意：保存先はブラウザ単位です。別のブラウザ・別のPC・シークレットモードでは設定も画像も引き継がれません。 またブラウザの「閲覧データの削除」でサイトデータを消すと、保存内容も消えます。

6. 利用上の注意（自分のPCで楽しむアプリです）

商品サムネイルなどの画像には著作権があります。 このアプリはWebへの公開はせず、自分のPCにダウンロードして、個人で楽しむ範囲でご利用ください。

プライバシーについて（安心して使える点）

  • アップロードした画像はどこにも送信されない — 画像や設定は自分のブラウザの localStorage に保存されるだけで、PCの外には出ない
  • 外部への通信が一切ない — 外部CDN・解析ツール・APIを使っていないため、勝手にどこかへアクセスすることがない
  • ネット接続不要 — ファイルをダブルクリックで開くだけで、オフラインでも動作する

8. 動作条件・制限事項まとめ

  • 必要環境：モダンブラウザのみ（Chrome / Edge / Firefox / Safari）。サーバー・ネット接続・インストール不要
  • 同フォルダに置くと表示が良くなるファイル：1000196889_2.png（ガチャボタン画像）、images/a.png〜（デフォルト画像）。なくても代替表示で動作する
  • 保存はブラウザ単位：別ブラウザ・別PC・シークレットモードでは設定も画像も引き継がれない
  • 履歴はシミュレートごとにリセット：記録を残したいときはCSV保存を使う
  • 種類数は最大24種：タイプ識別子はA〜Xの24文字。25種以上には現状対応していない
